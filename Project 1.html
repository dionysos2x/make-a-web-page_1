Lesson 1 - The Basics of the Web and HTML

The Basics of the Web

The world wide web (www) is basically a huge collection of HTML documents, located on many different computers which communicate via HTTP (HyperText Transfer Protocol), the main protocol of the web. The many HTML files are connected via hyperlinks (URLs). Web browsers can interpret the HTML files and display them for the user in a more appealing way. The web was invented in the 1990s and today consists of about 30 Billion pages.

Why Computers are Stupid

Computers are very sensitive to typos because they execute the programmers code literally and they simply do not understand wrong syntax and produce an error. A human instead could just correct the typo and would still be able to understand the code.
A computer is just as smart as the software it is running.

HTML

HTML stands for HyperText Markup Language.
It is made up of:
- text content (what you see)
- markup (describes how the text should look like)
- references to other documents (e.g. images, videos and music)
- links to other pages

Elements and Tags

HTML code is made of elements which are defined by tags. There are many different HTML tags e.g. markup tags to make text bold or italic, the anchor tag to create links like www.udacity.com
or the image tag to include images like this cute fellow . Other tags are for creating whitespace, like linebreaks
or paragraphs.
Most HTML elements are defined by an opening tag and a closing tag, but some elements don't need a closing tag. Those are called void tags.

Attributes

HTML Attributes are found inside tags and give the tag more variety and options. The anchor tag's "href"-attribute for example is needed to define the URL for a link to another webpage.

Inline and Block Elements

HTML elements can be categorized as "inline" or "block", where block elements form an "invisible box" around their content e.g. the paragraph tag or the div and span tags.

The HTML Document Structure

In general the structure of an HTML document is as follows:
- Definition of the doctype (doctypes bring different presettings for the HTML document)
- Head (title and implementation of CSS and JavaScript can be defined here)
- Body (the main content of the webpage)

Lesson 2 - Creating a Structured Document

Browser Developer Tools

The developer tool of a browser gives the opportunity to have a look at and play around with the sourcecode of a webpage while you can see changes immediately. By marking an element in the code, the associated object on the webpage will be highlighted.

HTML and DOM

HTML is (as any other language) defined by a syntax and certain rules. When the sourcecode of a webpage is viewed inside the developer tool of a browser, the HTML elements form a tree-like structure, because elements can include other elements. This structure is called DOM tree, where DOM (Document Object Model) is a language-independent convention for representation of and interaction with objects in HTML (and other markup languages).

CSS

CSS stands for Cascading Style Sheets and is another language which is used inside HTML to describe the look and formatting of HTML elements.

HTML Boxes

All objects on a webpage (text, pictures, icons etc.) are structured into boxes, even those which appear to be round or in any other not-even-close-to-square shape. Often those boxes are divided into smaller subboxes and so on. This "boxifying" gives the designer the opportunity to easily change parts of the page without affecting others. To create boxes, the HTML div and span tags can be used. Boxes are labeled with the "class" attribute.

Lesson 3 - Adding CSS Style and HTML Structure

Closer Look at CSS

The word cascading in Cascading Style Sheets is related to the tree-like nature of CSS where child elementsinherit all properties from parent elements. It is possible to apply CSS style to single elements, a group of elements or even all elements of the same tag.

This is a great source for all the different CSS properties: https://developer.mozilla.org/en-US/docs/Web/CSS/Reference.

CSS code is provided in a different file apart from the main *.html file and the implementation of that file happens in the HTML head. It is possible to implement CSS code inside HTML code but this is considered bad programming. It leads to repetition of code which we will avoid at all costs because repetition increases the propability for mistakes and also makes the file bigger and more confusing. It is also much faster to change something when you just have to change the code at one point.

Boxes Revisited

Every HTML box consists of four elements which are called from outside to inside: margin, border, padding and content. Each of these elements can be adjusted individually and should be defined in percentage rather than pixels because of different display sizes. This demo page gives a good visual example of the HTML box structure.

Code, Test, Refine

1. Look for natural boxes in your design.
2. Look for repeated styles and semantic elements.
3. Write your HTML.
4. Apply your styles (CSS).
5. Fix things.
Repeat steps 3 and 4 until it looks like your design mark.
